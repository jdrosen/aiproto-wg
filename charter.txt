The AI Agent Protocols (aiproto) Working Group is about defining a protocol framework enabling various interactions between AI Agents over a secure multiplexed transport. An AI Agent is an autonomous, adaptive intelligent software system that uses AI models to complete a specific task. To complete the task, the AI Agent may need to interact with external resources on the Internet. It may invoke tools - which are typically APIs that allow the AI Agent to obtain information or perform actions. It may also interact with other AI Agents, delegating tasks to them.

With the expansion of communication over the Internet between AI Agents and external resources (tools, AI Agents), communication reliability across platforms and vendors becomes more and moreincreasingly important. The role of the aiproto working group is to facilitate such interoperability.

Several considerations arise which are unique to AI Agent protocols, which are a focus of activity for protocol development:

* AI Agents offer differentiated skills to support task collaboration, which introduces new considerations regarding how AI Agents can be dynamically discovered by their skills for cross-domain interactions.

* Interactions of AI Agents with other AI Agents tools can be long lived, make use of the exchange of significant amounts of multi-modal context and require very low latency. This introduces new considerations around reliability, transport session management and data transport.

* To protect data exchanged between AI Agents (and between AI Agents and tools) over potentially untrusted networks, particularly when handling sensitive information (such as personal data or conversational context), mechanism are required to ensure confidentiality, integrity, and authenticity. This introduces new considerations around protocol-level security mechanisms.

The working group will produce the following deliverables:

Groundwork may bewill be documented through a set of informational Internet-Drafts, which include the following key components:
* Use cases focusing on Agent-to-agent communications, Agent-to-tool communications.

* A gap analysis and requirements document that examines existing open-source protocols, and derives protocol requirements related to discovery, Agent-to-agent communications, Agent-to-tool communications, and security aspects.

A standard track overall AI Agent protocol framework overview: 
This work encompasses the various building blocks and their interactions, covering AI Agent and tool discovery, Agent-to- agent/Agent-to-tool multi-modal communications, long-lived session management, and security aspects.

This protocol framework will enable the AI Agents to collaborate with other AI Agents on the Internet (deployed in various interconnected domains and ecosystem) to execute simple or complex tasks. It will allow multi-modal collaboration, i.e., using varied data formats such as text, images, video, audio, and structured data multi-modal contexts It will provide basic lifecycle management, enabling the establishment, update, and termination of the services of the downstream agent.

The aiproto Working Group will describe the framework that uses existing protocols and their extensions to achieve the definition of a protocol suites for Agent-to-agent and Agent-to-tool interoperable interactions. During this work, new protocol requirements may be defined, leading to potential new protocol work. The framework can get inspiration on existing agent communication protocols and frameworks (such as the A2A protocol currently or MCP protocols specified under the auspices of the Linux Foundation).

The aiproto working group is expected to closely coordinate with other related IETF working groups, such as Web Authorization Protocol(OAuth), webbotauth, WIMSE, on security considerations, WebTransport, MoQ, QUIC, TSVWG on data transport and Intarea for discovery. If the working group needs any changes to or extensions of protocols specified by other working groups, those issues will be raised with the relevant working groups for decisions on how best to handle them. The group is also expected to have communication with opensource projects running under Linux Foundation.
